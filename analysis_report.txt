1. ОПИСАНИЕ ПРОЕКТА
   Веб-сайт "Ліцей №167" — державний заклад освіти в Києві з поглибленим вивченням німецької мови.
   Односторінковий додаток (SPA) на React.

2. ТЕХНОЛОГІЧНИЙ СТЕК
   - Frontend: Vite + React 18 + TypeScript
   - Стилі: Tailwind CSS + PostCSS
   - Іконки: Lucide React
   - Бекенд: Firebase (Auth, Firestore, Storage)
   - Адмінка: Supabase (CRUD новин/відгуків)
   - Контент: JSON-файли + Markdown (зберігаються прямо в репо)
   - Хостинг: Netlify (SPA redirects + Netlify Forms)

4. ФУНКЦІОНАЛЬНІСТЬ
   - SPA-навігація без react-router (ручне перемикання сторінок через state в App.tsx)
   - Новини зберігаються як Markdown-файли в /content/news, підтягуються через GitHub API
   - Контент сторінок у JSON-файлах (локально або з GitHub raw)
   - Firebase: підключено Auth, Firestore, Storage (але поки не використовується активно)
   - Supabase: CRUD для новин (news_articles) та відгуків (reviews)
   - Netlify Forms: контактна форма
   - Адмінка: логін через Firebase, управління новинами/відгуками через Supabase
   - Відгуки статичні (захардкожені в Reviews.tsx)

5. ОСОБЛИВОСТІ
   - Firebase API ключ видимий у firebase.js (публічний, це нормально для Firebase)
   - Сторінка "Відкритість" (openup) містить багато посилань на Google Docs/Drive
   - Сімейне навчання (family) — розділ з акордеоном, але URL більшості класів пусті
   - Контент сторінок підтягується з GitHub raw (кешування браузера)
   - Є Netlify CMS конфіг (public/admin/config.yml та public/admin/index.html)

6. ЗАУВАЖЕННЯ
   - Сторінка "pages" в адмінці позначена як "скоро буде доступна"
   - У Reviews.tsx відгуки захардкожені, але в адмінці є CRUD для reviews (Supabase)
   - Дублювання логіки парсингу Markdown між News.tsx та NewsPost.tsx
   - В family.json багато пустих URL, потребує наповнення
